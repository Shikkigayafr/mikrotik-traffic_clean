## 📌 Описание проекта

Разработана система автоматического сбора и анализа интернет-трафика с маршрутизаторов MikroTik (модель RB760iGS) с последующей отправкой данных в Zabbix и формированием ежедневных и месячных отчётов.

**Ключевая задача:** Рассчитать **чистый интернет-трафик** (без учёта L2TP-туннелей), чтобы видеть реальную нагрузку на внешний канал.

---

## 🧩 Технические сложности

| №                 | Проблема |                                                 Решение |
|---|----------|---------|
| 1 | **Неправильные данные в Zabbix**          | Изначально пытался брать метрики из трендов Zabbix — там была неверная информация. Перешёл на прямой сбор через SNMP. |
| 2 | **Нестандартное поведение MikroTik**      | На модели RB760iGS счётчики хранятся в **оперативной памяти**. При перезагрузке роутера все данные сбрасываются. |
| 3 | **32-битные счётчики**                    | Счётчики имеют лимит `4 294 967 295` байт (~4 ГБ). При превышении происходит переполнение. Скрипт обрабатывает этот случай. |
| 4 | **Трафик L2TP идёт через WAN**            | Нужно было отделить "чистый интернет" от трафика L2TP-туннелей, чтобы видеть реальную нагрузку на внешний канал. |
| 5 | **Потеря данных при недоступности**       | Если роутер не отвечает, скрипт не должен искажать дневную статистику. |

## 🛠️ Архитектура решения

```text
Скрипт запускается каждые 10 минут (cron)
         ↓
SNMP-запрос к MikroTik (ECM и SM)
         ↓
Получение значений счётчиков (WAN, L2TP)
         ↓
Сравнение с предыдущими значениями (state-файлы)
         ↓
Расчёт дельты за 10 минут
         ↓
Вычитание L2TP из WAN → чистый интернет-трафик
         ↓
Сохранение в дневной файл (daily_YYYYMMDD.txt)
         ↓
Отправка в Zabbix через zabbix_sender (бит/с)
         ↓
Ежедневный скрипт формирует месячный CSV-отчёт 
```
# Структура скриптов

### 1. `snmp_traffic_collector.sh` — основной сборщик

- Автоматическое определение индексов L2TP-туннелей
- Сбор данных с ECM (192.168.1.1) и SM (192.168.0.1)
- Обработка переполнения 32-битных счётчиков
- Расчёт чистого трафика = WAN - L2TP
- Накопление дневных сумм в файле `daily_YYYYMMDD.txt`
- Отправка данных в Zabbix через `zabbix_sender`
- Логирование всех действий

### 2. `snmp_daily_summary.sh` — формирование отчётов

- Чтение дневного файла за вчерашний день
- Перевод байтов в гигабайты
- Добавление записи в месячный CSV-отчёт (`traffic_YYYYMM.csv`)
- Удаление обработанного дневного файла

## 🔧 Интеграция с Zabbix

Данные отправляются в Zabbix через `zabbix_sender`:

```bash
zabbix_sender -z 127.0.0.1 -s "host" -k "clean.traffic.@название.in" -o "$CLEAN_@название_BPS"
```

# Установка и настройка.
### 1. Скопируй скрипты на сервер

Выполни на своей локальной машине (или там, где лежат скрипты):

```bash
scp scripts/snmp_traffic_collector.sh user@zabbix-server:/home/user/scripts/
scp scripts/snmp_daily_summary.sh user@zabbix-server:/home/user/scripts/
```
# 💡 Примечание:

user — твой логин на сервере

zabbix-server — IP-адрес или доменное имя сервера

/home/user/scripts/ — путь к папке, куда кладёшь скрипты (можешь использовать любой другой)

### 2. Сделай скрипты исполняемыми

Зайди на сервер и выполни:

```bash
chmod +x /home/user/scripts/snmp_traffic_collector.sh
chmod +x /home/user/scripts/snmp_daily_summary.sh
```
# 💡 Примечание:

Убедись, что путь к скриптам соответствует тому, куда ты их скопировал.

Без прав на выполнение (+x) скрипты не запустятся через cron.

### 3. Настрой автоматический запуск через cron

Открой файл с задачами cron:

```bash
crontab -e

Добавь в конец файла две строки:

# Сбор данных каждые 10 минут
*/10 * * * * /home/user/scripts/snmp_traffic_collector.sh

# Подведение итогов дня в 23:55
55 23 * * * /home/user/scripts/snmp_daily_summary.sh
```
Что это даёт:

Время	Действие	Результат
Каждые 10 минут	Запуск snmp_traffic_collector.sh	Данные в Zabbix обновляются каждые 10 минут
Каждый день в 23:55	Запуск snmp_daily_summary.sh	Утром ты видишь отчёт за вчерашний день

# 💡 Примечание:

Если хочешь изменить интервал сбора — отредактируй */10 на нужное значение (например, */5 — каждые 5 минут).

Время подведения итогов можно изменить, поменяв 55 23 на нужное время (например, 00 00 — в полночь).

### 4. Проверь, что всё работает

Посмотри логи в реальном времени:

```bash
tail -f /home/user/logs/snmp_collector.log

Если видишь строки вида:

2026-06-17 14:30:01 - Собрано: @ чистый = 125 МБ, @ чистый = 89 МБ
2026-06-17 14:30:02 - Zabbix: @ = 125000 bps

→ всё работает! 🎉

Если ошибки — проверь:

Доступность роутеров по сети (ping 192.168.1.1)

Наличие утилиты zabbix_sender (which zabbix_sender)

Права на запись в папки snmp_state и snmp_data

💡 Примечание:

Логи пишутся в файл, путь к которому указан в скрипте (LOG_FILE).

Если логов нет — проверь, что скрипт вообще запускается (выполни его вручную и посмотри вывод).

```

### 5. Проверь дневной отчёт

На следующий день проверь, что отчёт сформировался:

```bash
ls -la /home/user/reports/

Открой свежий CSV-файл:

bash
cat /home/user/reports/traffic_$(date +%Y%m).csv
Пример вывода:

text
date;wan_@_gb;l2tp_@_gb;clean_@_gb;wan_@@_gb;l2tp_@@_gb;clean_@@_gb
20260616;12.345;2.100;10.245;8.900;1.200;7.700
20260617;14.200;2.500;11.700;9.300;1.400;7.900
Если файл есть и в нём есть данные → всё настроено правильно ✅
Если файла нет → проверь, что ежедневный скрипт запустился (смотри логи в snmp_collector.log)

💡 Примечание:

Отчёты сохраняются в папку REPORT_DIR, путь задаётся в snmp_daily_summary.sh.

Если ты хочешь изменить формат или место хранения — отредактируй переменную REPORT_DIR в скрипте.
```
### 6. Что дальше

После завершения всех шагов система работает в автоматическом режиме:

- Скрипты запускаются по расписанию (cron)
- Данные собираются и передаются в Zabbix
- Отчёты формируются и сохраняются в CSV

> **💡 Рекомендации:**
> - Раз в месяц проверяй, что логи не разрастаются (можно настроить ротацию)
> - Если показания трафика перестали приходить — первым делом проверь доступность роутера по SNMP
> - При изменении IP-адресов роутеров или имён L2TP-интерфейсов — обнови соответствующие переменные в скрипте
